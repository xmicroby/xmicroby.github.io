**Data model — define these interfaces and constants at the top:**

```typescript
interface WorkflowStep {
  id: string;
  name: string;
  description: string;
  icon: JSX.Element; // simple inline SVG, 20x20 viewBox
  loop: 'design' | 'workbench' | 'optimize' | null; // which loop this step belongs to
}

interface Loop {
  id: 'design' | 'workbench' | 'optimize';
  label: string;
  color: string; // CSS custom property — ALL THREE use var(--color-accent) per UI-SPEC (accent reserved for active states)
  steps: string[]; // step IDs in this loop
}
```

**Workflow steps (7 nodes in the DAG):**
1. `chat` — "Chat with AI" — "Upload a clinical registry spec document and describe your extraction needs. The AI agent reads the spec and proposes a pipeline." — Loop: design — Icon: chat bubble SVG
2. `generate` — "Generated Pipeline" — "The AI produces a validated pipeline definition as a directed acyclic graph (DAG) — each node is an LLM extraction step with typed inputs and outputs." — Loop: design — Icon: graph/flowchart SVG
3. `test` — "Run Ad-Hoc Test" — "Upload a sample clinical document and run the pipeline against it. See extracted values for each field in real time." — Loop: workbench — Icon: play/run SVG
4. `review` — "Review Output" — "Inspect extracted values alongside the source text. Accept correct results or flag errors for correction." — Loop: workbench — Icon: check/magnify SVG
5. `label` — "Label Corrections" — "Corrected extractions automatically become labeled training examples — no manual annotation workflow needed." — Loop: optimize — Icon: tag/label SVG
6. `optimize` — "DSPy Optimize" — "Labeled corrections feed into DSPy GEPA optimization, which refines each extraction node's prompt and few-shot examples." — Loop: optimize — Icon: sparkle/tune SVG
7. `deploy` — "Improved Pipeline" — "The optimized pipeline version is saved with full audit trail. Accuracy improves incrementally with each correction cycle." — Loop: null (terminal) — Icon: rocket/ship SVG

];

**Layout — render as a structured vertical flow with loop groupings:**
- Render the 7 steps in a vertical layout with clear loop groupings
- Each loop is a visual group with a dashed border (`border-dashed border-[var(--color-border)]`) and its loop label shown as a small monospace tag at the top-left: `font-mono text-xs text-[var(--color-accent)]`
- Within each loop group, steps are arranged horizontally (2 nodes side by side) with a curved arrow between them indicating the cycle
- A forward arrow connects each loop group to the next (Design → Workbench → Optimize → deploy)
- The `deploy` node sits outside any loop group at the bottom

**Auto-cycle animation (per D-06):**
- State: `activeLoop: 'design' | 'workbench' | 'optimize' | null` (which loop is currently highlighted)
- State: `userSelected: boolean` (true if user clicked a loop; false = auto-cycling)
- On mount, start a `setInterval` cycling through loops every 3 seconds: design → workbench → optimize → design → ...
- When a loop is active, its group border animates to `var(--color-accent)` with a subtle pulse (opacity animation between 0.5 and 1.0), and the loop label text becomes bold
- Non-active loop groups remain at default border (`var(--color-border)`)

**Click-to-isolate loop (per D-06):**
- Clicking a loop group sets `activeLoop` to that loop and `userSelected = true` (stops auto-cycle)
- Clicking the same loop again (or clicking outside all loops) sets `userSelected = false` (restarts auto-cycle)
- Use a `ref` on the container and a click handler that checks if the click target is inside any loop group

**Click-to-expand step nodes (per D-07):**
- State: `expandedStep: string | null`
- Clicking a step node toggles its description (same AnimatePresence pattern as PipelineDiagram)
- Step expand and loop highlight are independent interactions (expanding a step within a non-active loop doesn't change the active loop)

**Step node rendering — follow PipelineDiagram pattern exactly:**
- `<button type="button" aria-expanded={expandedStep === step.id}>`
- Card: `bg-[var(--color-surface-elevated)] border border-[var(--color-border)]` (expanded: `border-[var(--color-accent)]`)
- Step label: `font-mono text-xs`, step number prefix in accent color
- Description: `AnimatePresence` with height/opacity animation, `aria-live="polite"`
- Icons: inline SVG, 20x20 viewBox, monochrome via `currentColor`, `aria-hidden="true"`

**Accessibility:**
- `useReducedMotion()` — if true: show ALL loops highlighted statically (no cycling), no pulse animation, all step descriptions visible, no height animations
- All step nodes are `<button>` elements with `aria-expanded`
- Loop groups have `role="group"` with `aria-label="Design Loop"` etc.
- The overall component has `role="figure"` with `aria-label="ChartExtract pipeline builder workflow showing three iterative development loops"`

**Size target:** ~250-350 lines. This is the largest component — justify the size with the three independent interaction systems (auto-cycle, loop click, step expand).
