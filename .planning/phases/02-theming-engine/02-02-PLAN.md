**Known issues to check and fix:**

1. **Header backdrop-blur tint (Header.astro):**
   - Current: `bg-[var(--color-surface)]/85` — 85% opacity of surface color
   - In dark mode: `rgba(19,26,43,0.85)` over dark bg → fine
   - In light mode: `rgba(255,255,255,0.85)` over light bg → should be fine, but verify the blur effect is still visible. If the header blends too much into the page background in light mode, adjust opacity to `/90` or `/80`.
   - Fix only if needed after visual inspection of the built output.

2. **MobileNav overlay (MobileNav.astro):**
   - Current: `bg-black/50` — this works in dark mode. In light mode, black/50 overlay is standard. No change expected, but verify.

3. **Hero CTA button (index.astro):**
   - Current: `border border-[var(--color-accent)]` with accent text
   - Light mode accent is `#0d9488` (teal) on `#f8fafb` bg — good contrast (WCAG AA passes)
   - Dark mode accent is `#4ecdc4` on `#0a0e17` — good contrast. No change expected.

4. **Section accent labels (Section.astro):**
   - `text-[var(--color-accent)]` — light `#0d9488` on `#f8fafb` = sufficient contrast. Dark `#4ecdc4` on `#0a0e17` = good. No change expected.

5. **Body fade-in animation (global.css):**
   - The `fadeIn` animation on body may conflict with the theme transition system. The `theme-ready` gate should handle this, but verify the animation doesn't cause a white flash in dark mode. If it does, change the animation to only animate `opacity` (which it already does) and ensure the FOUC script runs before the animation starts.

6. **Add `color-scheme` CSS property to global.css** for native browser UI elements (scrollbars, form controls):
```css
:root {
  color-scheme: light;
  /* existing tokens... */
}
.dark {
  color-scheme: dark;
  /* existing tokens... */
}
```
This tells the browser to render native UI elements (scrollbar, text selection, form inputs) in the appropriate theme. Add `color-scheme: light;` as the first line inside the `:root` block and `color-scheme: dark;` as the first line inside the `.dark` block.

After fixing any issues found, run `npm run build` and open the built output in a browser to verify both themes render correctly. Use browser dev tools to toggle the `.dark` class on `<html>` to compare.
