    ```astro
      <script>
        const sections = document.querySelectorAll('section[id]');
        const navLinks = document.querySelectorAll('.nav-link[data-section], .mobile-nav-link[data-section]');

        const observer = new IntersectionObserver(
          (entries) => {
            entries.forEach((entry) => {
              if (entry.isIntersecting) {
                const sectionId = entry.target.getAttribute('id');

                navLinks.forEach((link) => {
                  const linkSection = link.getAttribute('data-section');
                  if (linkSection === sectionId) {
                    link.classList.add('text-[var(--color-accent)]');
                    link.classList.remove('text-[var(--color-text-muted)]');
                  } else {
                    link.classList.remove('text-[var(--color-accent)]');
                    link.classList.add('text-[var(--color-text-muted)]');
                  }
                });
              }
            });
          },
          {
            rootMargin: '-80px 0px -50% 0px',
            threshold: 0,
          }
        );

        sections.forEach((section) => observer.observe(section));
      </script>
    </BaseLayout>
    ```

    - IntersectionObserver watches all `section[id]` elements, highlights the corresponding `nav-link` and `mobile-nav-link` when a section enters the viewport
    - Observer `rootMargin: '-80px 0px -50% 0px'` accounts for sticky header (80px) and triggers when section is in top half of viewport
    - The script is inline (not a module) per Astro zero-JS philosophy — it runs directly without bundling

  <action>
    Create `.github/workflows/deploy.yml` using the official Astro GitHub Pages deployment workflow (per D-13):

    ```yaml
    name: Deploy to GitHub Pages

    on:
      push:
        branches: [main]
      workflow_dispatch:

    permissions:
      contents: read
      pages: write
      id-token: write

    concurrency:
      group: "pages"
      cancel-in-progress: false

    jobs:
      build:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4

          - name: Install, build, and upload
            uses: withastro/action@v3
            # with:
            #   path: . # Root of Astro project (default)
            #   node-version: 22 # Node version (default: 22)
            #   package-manager: npm@latest # (default: auto-detect)

      deploy:
        needs: build
        runs-on: ubuntu-latest
        environment:
          name: github-pages
          url: ${{ steps.deployment.outputs.page_url }}
        steps:
          - name: Deploy to GitHub Pages
            id: deployment
            uses: actions/deploy-pages@v4
    ```

    This workflow:
    - Triggers on push to `main` branch (per D-14) and manual dispatch
    - Uses `withastro/action@v3` which handles `npm install` + `astro build` + artifact upload
    - Uses `actions/deploy-pages@v4` for GitHub Pages deployment via artifacts (not gh-pages branch, per D-14)
    - Sets proper permissions for Pages deployment
    - Concurrency prevents overlapping deployments

    Verify the YAML is valid by checking syntax.
