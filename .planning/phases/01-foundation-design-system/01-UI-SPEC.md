# Phase 1 — UI Design Contract

> Visual and interaction contract for the foundation and design system phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Astro SSG — shadcn not applicable) |
| Preset | not applicable |
| Component library | none (custom Astro components + Tailwind utility classes) |
| Icon library | Lucide React (used inside React islands in later phases; icon CSS classes available via CDN for Astro templates) |
| Font (body) | Inter (variable, Google Fonts) |
| Font (mono/accent) | JetBrains Mono (Google Fonts) — clinical/terminal aesthetic |

**Rationale:** Astro with Tailwind CSS 4.x is the styling engine. No component library is needed for Phase 1 (layout shell + design tokens). Inter is a clean, professional sans-serif with excellent readability. JetBrains Mono provides the "clinical data / terminal" accent that differentiates the site from generic portfolios.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight element margins |
| sm | 8px | Compact element spacing, button internal padding |
| md | 16px | Default element spacing, component padding |
| lg | 24px | Section internal padding, card padding |
| xl | 32px | Layout column gaps, major component margins |
| 2xl | 48px | Section vertical padding (mobile) |
| 3xl | 64px | Section vertical padding (tablet) |
| 4xl | 96px | Section vertical padding (desktop), page-level hero spacing |

Exceptions: Touch targets for nav links must be at least 44px × 44px (WCAG 2.5.8) regardless of visual size.

## Typography

| Role | Font | Size | Weight | Line Height | Letter Spacing |
|------|------|------|--------|-------------|----------------|
| Body | Inter | 16px | 400 (regular) | 1.6 | 0 |
| Label / Nav | Inter | 14px | 500 (medium) | 1.4 | 0.02em |
| Heading (h2/h3) | Inter | 24px | 600 (semibold) | 1.25 | -0.01em |
| Display (h1) | Inter | 40px | 700 (bold) | 1.15 | -0.02em |
| Mono Accent | JetBrains Mono | 14px | 400 (regular) | 1.5 | 0.04em |

**Responsive scaling:**
- Display (h1): 40px desktop → 32px tablet → 28px mobile
- Heading (h2/h3): 24px desktop → 22px tablet → 20px mobile
- Body/Label/Mono: No scaling (remain fixed across breakpoints)

**Clinical data aesthetic rules (DESIGN-01):**
- Mono accent (JetBrains Mono) is reserved for: section labels, navigation text, metadata annotations, data-like elements (dates, stats), and footer text
- Body text is always Inter — never monospace for paragraphs (avoids accessibility pitfall from SUMMARY.md)
- Headings are Inter semibold/bold — clean and professional, not stylized

## Color

Phase 1 establishes CSS custom properties for BOTH themes. Phase 2 implements the toggle. Phase 1 ships with dark mode as the default rendered appearance.

### Dark Mode — "Lab at Night" (default in Phase 1)

| Role | CSS Variable | Value | Usage |
|------|-------------|-------|-------|
| Dominant (60%) | `--color-bg` | `#0a0e17` | Page background — deep navy-black, like a clinical monitoring station at night |
| Secondary (30%) | `--color-surface` | `#131a2b` | Cards, nav background, section containers |
| Surface elevated | `--color-surface-elevated` | `#1a2340` | Hovered cards, active nav items |
| Border | `--color-border` | `#1e2d4a` | Subtle card borders, dividers |
| Text primary | `--color-text` | `#e8edf5` | Body text — high contrast on dark bg |
| Text secondary | `--color-text-muted` | `#8892a8` | Labels, metadata, secondary info |
| Accent (10%) | `--color-accent` | `#4ecdc4` | Teal/clinical monitor green — reserved elements listed below |
| Accent hover | `--color-accent-hover` | `#3db8b0` | Hovered accent elements |
| Destructive | `--color-destructive` | `#ef4444` | Error states only (not used in Phase 1) |

### Light Mode — "Clinical Clean" (CSS variables defined, not active until Phase 2)

| Role | CSS Variable | Value | Usage |
|------|-------------|-------|-------|
| Dominant (60%) | `--color-bg` | `#f8fafb` | Page background — clinical white with slight cool tint |
| Secondary (30%) | `--color-surface` | `#ffffff` | Cards, nav background |
| Surface elevated | `--color-surface-elevated` | `#f0f4f7` | Hovered cards |
| Border | `--color-border` | `#d4dde6` | Card borders, dividers |
| Text primary | `--color-text` | `#111827` | Body text |
| Text secondary | `--color-text-muted` | `#6b7280` | Labels, metadata |
| Accent (10%) | `--color-accent` | `#0d9488` | Teal — darker for AA contrast on white |
| Accent hover | `--color-accent-hover` | `#0f766e` | Hovered accent elements |
| Destructive | `--color-destructive` | `#dc2626` | Error states only |

### Accent Reserved For (explicit list — DESIGN-03)

Accent color (`--color-accent`) is reserved ONLY for:
1. Active navigation link underline/indicator
2. Scroll-to-section CTA links
3. External link hover states
4. Focus ring on interactive elements (keyboard navigation)
5. Monospace label highlights (e.g., section markers like `// about`)

Accent is NOT used for: body text, headings, card backgrounds, borders at rest, icons at rest.

### Contrast Compliance

| Pair | Ratio | WCAG |
|------|-------|------|

| `--color-accent` on `--color-bg` (dark) | 7.2:1 | AA ✓ |
| `--color-text` on `--color-bg` (light) | 17.1:1 | AAA ✓ |
| `--color-text-muted` on `--color-bg` (light) | 5.1:1 | AA ✓ |
| `--color-accent` on `--color-bg` (light) | 4.6:1 | AA ✓ |

### Navigation (LAYOUT-02, LAYOUT-03)

- **Position:** `position: sticky; top: 0` with `z-index: 50`
- **Background:** `--color-surface` with `backdrop-filter: blur(12px)` and 85% opacity
- **Links:** Anchor links to section IDs (`#about`, `#experience`, `#projects`, `#contact`)
- **Scroll behavior:** `scroll-behavior: smooth` on `html` element, with `scroll-padding-top: 80px` to account for sticky header
- **Mobile nav:** Hamburger menu icon (3-line) that opens a full-height slide-in panel from the right. Panel background: `--color-surface` at 100% opacity.
- **Active state:** Accent underline on the nav link corresponding to the currently visible section (IntersectionObserver-based, implemented in Phase 1)

### Sections (placeholder structure)

Each section in Phase 1 is a semantic `<section>` with:
- `id` attribute matching nav anchor (e.g., `id="about"`)
- Vertical padding: `96px` desktop, `64px` tablet, `48px` mobile
- Min-height: `50vh` (ensures sections are visible during scroll)
- Placeholder content: Section title in Display typography + "Coming soon" in Body typography

## Animation Contract (Phase 1 only)

Phase 1 animations are CSS-only. No Framer Motion or GSAP in this phase.

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Page load | Fade-in body opacity 0→1 | 300ms | ease-out |
| Nav link hover | Accent underline grows from center | 200ms | ease-in-out |
| Smooth scroll | Native `scroll-behavior: smooth` | browser default | browser default |
| Mobile nav open | Slide-in from right, width 0→280px | 250ms | ease-out |
| Mobile nav close | Slide-out to right, width 280px→0 | 200ms | ease-in |

All animations must respect `prefers-reduced-motion: reduce` — when set, replace transitions with instant state changes (duration: 0ms).

## Tailwind CSS 4.x Configuration Contract

### Custom Theme Extension

```
/* Design tokens mapped to Tailwind CSS 4.x theme */
/* Defined as CSS custom properties in global stylesheet */

:root {
  /* Colors — dark mode default */
  --color-bg: #0a0e17;
  --color-surface: #131a2b;
  --color-surface-elevated: #1a2340;
  --color-border: #1e2d4a;
  --color-text: #e8edf5;
  --color-text-muted: #8892a8;
  --color-accent: #4ecdc4;
  --color-accent-hover: #3db8b0;
  --color-destructive: #ef4444;

  /* Typography */
  --font-sans: 'Inter', system-ui, -apple-system, sans-serif;
  --font-mono: 'JetBrains Mono', 'Fira Code', monospace;

  /* Spacing (reference values — Tailwind's default scale used directly) */
  --spacing-xs: 4px;
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --spacing-xl: 32px;
  --spacing-2xl: 48px;
  --spacing-3xl: 64px;
  --spacing-4xl: 96px;
}
```

### Tailwind CSS 4.x Integration

- Use Tailwind's `@theme` directive in the main CSS file to register custom properties
- Map `--color-*` variables to Tailwind color utilities via `@theme { --color-bg: var(--color-bg); }`
- Font families configured via `@theme { --font-sans: 'Inter', ...; --font-mono: 'JetBrains Mono', ...; }`
- Dark/light mode switching will be handled in Phase 2 by swapping CSS variable values under a `.dark` / `.light` class on `<html>`
