  <action>
    Initialize Astro project in the repo root (not a subdirectory) using npm:

    1. Run `npm create astro@latest . -- --template minimal --install --no-git --typescript strict` to scaffold into current directory. If interactive prompts appear, use `--yes` flag or pipe `yes` to accept defaults.
       - If `npm create astro@latest` fails or prompts interactively, manually create the minimal structure:
         - `package.json` with `astro` dependency, scripts: `"dev": "astro dev"`, `"build": "astro build"`, `"preview": "astro preview"`
         - `astro.config.mjs` with `import { defineConfig } from 'astro/config'; export default defineConfig({});`
         - `tsconfig.json` extending `"astro/tsconfigs/strict"`
         - Create directories: `src/pages/`, `src/layouts/`, `src/components/`, `src/styles/`, `public/`

    2. Install Tailwind CSS 4.x for Astro:
       - Run `npm install @astrojs/tailwind tailwindcss@latest`
       - If `@astrojs/tailwind` is not compatible with Tailwind 4.x, use the Vite plugin approach instead:
         - `npm install tailwindcss@latest @tailwindcss/vite`
         - In `astro.config.mjs`: import `tailwindcss from '@tailwindcss/vite'` and add to `vite.plugins: [tailwindcss()]`

    3. Update `astro.config.mjs`:
       ```js
       import { defineConfig } from 'astro/config';
       import tailwindcss from '@tailwindcss/vite';

       export default defineConfig({
         site: 'https://youngbinkim.com',
         vite: {
           plugins: [tailwindcss()],
         },
       });
       ```

    4. Copy `CNAME` to `public/CNAME` with content `youngbinkim.com` (per D-15).

    5. Ensure `.gitignore` includes: `node_modules/`, `dist/`, `.astro/`, `.DS_Store`.

    6. Create a minimal `src/pages/index.astro` that just says `<html><body><h1>Hello</h1></body></html>` to verify the build works.

    7. Run `npm run build` to verify the Astro build completes without errors.
  </action>
