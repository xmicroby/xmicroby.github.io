    **Create `src/components/react/ScrollReveal.tsx`:**

    A reusable React component that wraps children with Framer Motion's staggered viewport-triggered animation.

    Architecture:
    ```tsx
    import { motion, useReducedMotion } from "motion/react";
    ```

    Props:
    - `children: React.ReactNode` — the content to animate
    - `staggerDelay?: number` — delay between children (default: 0.08s)
    - `className?: string` — optional wrapper class

    **Container variants (parent):**
    - `hidden: {}` (no styles — just a grouping container)
    - `visible: { transition: { staggerChildren: staggerDelay } }`

    **Item variants (each direct child):**
    - `hidden: { opacity: 0, y: 20, filter: "blur(2px)" }` — starts invisible, slightly below, with subtle blur for "data loading" effect (D-04)
    - `visible: { opacity: 1, y: 0, filter: "blur(0px)", transition: { duration: 0.5, ease: "easeOut" } }`

    **The component should:**
    1. Use `useReducedMotion()` hook from Motion
    2. If reduced motion: render children directly without any `motion` wrappers (pure passthrough)
    3. If animation enabled: render a `<motion.div>` parent with `initial="hidden"`, `whileInView="visible"`, `viewport={{ once: true, margin: "-50px" }}`
    4. Wrap each direct child in a `<motion.div variants={itemVariants}>` — use `React.Children.map` to wrap each child

    **Important design note:** The ScrollReveal component animates its direct children. This means the caller needs to pass each "item" as a separate child. For the About section, this means passing the grid columns as children. For Experience, passing the timeline wrapper.

    **Alternative simpler approach (RECOMMENDED):** Instead of wrapping individual children, make ScrollReveal a simple fade-up-stagger container. The parent fades/slides up as one unit. This is simpler and avoids the complexity of mapping over Astro-rendered children in a React island.

    Actually, the cleanest pattern for Astro islands is:
    - ScrollReveal wraps a SINGLE section's content and animates the whole block as one fade-up
    - For stagger WITHIN a section, create the individual items inside the React component OR use multiple ScrollReveal instances with increasing `delay` prop

    **RECOMMENDED FINAL APPROACH:**
    Make ScrollReveal accept a `delay` prop (default: 0) that adds an initial delay before the animation plays. Then in index.astro, wrap each major content block in its own `<ScrollReveal>` with incrementing delays:

    ```astro
    <Section id="about" label="// about" title="About">
      <ScrollReveal client:visible>
        <div class="grid md:grid-cols-2 ..."><!-- Who I Am --></div>
      </ScrollReveal>
    </Section>
    ```

    For staggered children within a section (like expertise cards or timeline stops), the stagger can be done with CSS `transition-delay` applied via Tailwind's `[style]` binding, since the parent ScrollReveal triggers the visibility. Each child gets:
    - `opacity-0` initially (via a CSS class)
    - When ScrollReveal's `whileInView` fires, it adds a class or the children inherit the animation

    **SIMPLEST CORRECT APPROACH (use this):**
    ScrollReveal is a motion.div that:
    1. Fades up its entire children block (`opacity: 0→1, y: 20→0`)
    2. Uses `whileInView` with `once: true`
    3. Accepts a `delay` prop to stagger between sections
    4. Respects reduced motion

    In index.astro, each section's inner content gets wrapped in `<ScrollReveal client:visible delay={N}>`.
    Use delay=0 for About's "Who I Am", delay=0.1 for "What I Do", delay=0 for Experience, delay=0.15 for each timeline column, etc.

    To get TRUE per-element stagger within sections (D-04/D-05 "data loading" effect), embed the section content INSIDE the React component where you have full control over `motion.div` children. But this means re-creating the HTML inside React, which duplicates markup.

    **PRAGMATIC DECISION:** Use ScrollReveal as a simple animated wrapper with a delay prop. The "stagger" effect comes from wrapping MULTIPLE blocks per section with incrementing delays. This avoids duplicating Astro HTML in React while still achieving the sequential pop-in effect (D-04, D-05).

    For the About section:
    ```astro
    <ScrollReveal client:visible delay={0}><!-- Who I Am block --></ScrollReveal>
    <ScrollReveal client:visible delay={0.15}><!-- What I Do block --></ScrollReveal>
    ```

    For the Experience section:
    ```astro
    <ScrollReveal client:visible delay={0}><!-- NYP stop --></ScrollReveal>
    <ScrollReveal client:visible delay={0.1}><!-- Columbia stop --></ScrollReveal>
    <ScrollReveal client:visible delay={0.2}><!-- Genentech stop --></ScrollReveal>
    <ScrollReveal client:visible delay={0.3}><!-- UC Berkeley stop --></ScrollReveal>
    ```

    This gives each timeline stop its own staggered entrance — exactly the "data loading" effect (D-04, D-05).

    **Implementation in index.astro:**

    2. In the About section, break the existing grid into two ScrollReveal wrappers (one for each column) with delays 0 and 0.15

    4. In the Projects section, wrap the content in a single ScrollReveal with delay 0
    5. Keep Section component's label and title OUTSIDE ScrollReveal — they should be visible immediately as static content. Only the section body content gets the reveal treatment.

    Actually, wrapping the section header (label + title) in its own ScrollReveal would be nice too — it should fade in first, then the content staggers in after. So:
    - Section label + title: wrapped in ScrollReveal with delay 0
    - Section body content: wrapped in ScrollReveal(s) with delay 0.1+

    BUT the Section component renders label/title — to wrap them, you'd need to modify Section.astro or duplicate the label/title in index.astro.

    **FINAL DECISION:** Don't modify Section.astro. Instead, wrap each Section's SLOT CONTENT (the part passed to Section) in ScrollReveal. The section label and title remain static (always visible). This is the least-invasive approach.
